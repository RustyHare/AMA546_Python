    编写Python程序的最佳实践

    三年前，当我写下这个的时候，只有上帝和我能够看懂。 现在，只有上帝能看懂了。
             ----某程序员

    然而，在实际编写程序的时候，忘记一个变量的任务、一个函数的作用或一个文件的含义用不了三年，一天足矣。为了给未来的自己以及有可能读到这些代码的人带来便利，也为了让代码看起来至少好看一些，人们总结出了一些实用的经验。学习这些经验，能够极大地提升编程效率和体验，让自己的代码变得美观且适宜阅读。

    命名规范

    # 使用拼音通常不是一个好想法。
nianling = 23
# 为什么呢？
def lianggejuzhenxiangcheng(juzhen1, juzhen2):
    pass

# 所以，在命名时，可以使用几个简单、直观且直接描述功能的单词命名：
def matrix_multiply(matrix1, matrix2):
    pass

# 在Python的社区中，常常会推荐以下的命名习惯。
# 这并非强制措施，但是绝大部分的大型公开包都或多或少遵循了它们。

# 变量命名：推荐使用小写字母+下划线
temp_matrix_1 = []

# 函数命名：使用小写字母+下划线
def matrix_multiply(matrix1, matrix2):
    pass

# 类命名：使用大写开头的单词。又称大驼峰命名法
class MyConvolutionalNeuralNetwork:
    ...

# 全局常量：使用全大写字母+下划线。某些IDE会给它们标上特殊颜色。
TRAIN_DEVICE = "cpu"

    代码分割

    python支持将代码分割为多个文件，可以从工程目录的其它.py文件中导入它定义的东西。比方说，在训练一个模型的时候，将网络结构写在一个单独的文件中，将损失函数写在另一个文件中，之后按需导入。
    比如，工程目录拥有如下的文件结构：
    ./loss.py
    ./model.py
    ./utils/utils.py
    ./main.py
    那么，在main.py中可以使用如下的导入方式来导入： 

    # main.py
from loss import CrossEntropyLoss
from model import DarkNet19
from utils.utils import learningRateUpdate

    * 自从Python 3.3开始，__init__.py不再是定义一个包所必需的标识。

    注释和文档

    我最讨厌两件事情：写注释，和看没有注释的代码。
             ----某程序员

    为一个类或函数编写注释是代码维护中非常重要的部分。
    注释在完成"清晰传达信息"的基础上，其格式可以自由发挥。不过，PEP-257推荐使用多行注释块。 

    """
对于一个.py文件的注释可以写在其顶端，推荐使用由三个连续双引号包裹的多行注释。
"""
def add(a, b):
    """可以使用简短的单行注释，但是仍然要多行注释块"""
    return a + b

def minus(a, b):
    """
    也可以使用多行。
    参数：
    a: int
    b: int
    返回值：
    返回a-b。
    """

    文档则是一个项目最先展示给阅读者的部分，一般放置在根目录的readme.md文件中。

    运算符与空格

    class SomeClass:
    # 冒号前面不应有空格，后面应当有空格
    label: str
    # 多个参数用逗号分隔，逗号后面应当有空格
    def some_method(a, b):
        # 加减乘除最好加空格
        return a + b
